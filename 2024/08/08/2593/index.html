LuckPerms 插件文档

介绍

LuckPerms 是一个高度灵活且功能强大的权限管理插件，适用于 Minecraft 服务器。它提供了精细的权限控制、多种数据存储选项和直观的用户界面，使得服务器管理员能够轻松管理玩家权限。

功能

  • 支持多种数据存储类型（MySQL、SQLite、H2、MongoDB等）
  • 提供详细的权限配置和组管理
  • 具有易于使用的 Web 界面
  • 支持多种平台（Bukkit, BungeeCord, Sponge, Nukkit, Velocity）
  • 轻量高效，适用于各种规模的服务器
  • 强大的API接口，便于开发者扩展和集成

安装

  1. 下载插件文件 LuckPerms.jar。
  2. 将 LuckPerms.jar 文件放入服务器的 plugins 文件夹中。
  3. 重启服务器。

配置

安装插件后，可以通过编辑 config.yml 文件来配置插件。以下是默认的配置示例：

配置选项

  • storage-method: 数据存储方式，可选 h2, mysql, sqlite, mongodb 等。
  • data.address: 数据库地址。
  • data.port: 数据库端口。
  • data.database: 数据库名称。
  • data.username: 数据库用户名。
  • data.password: 数据库密码。
  • server.server-name: 服务器名称。

命令

  • /lp user <user> permission set <permission> [value]: 设置玩家的权限。
  • /lp group <group> permission set <permission> [value]: 设置组的权限。
  • /lp user <user> parent add <group>: 将玩家添加到组。
  • /lp user <user> parent remove <group>: 将玩家从组中移除。
  • /lp group <group> create: 创建一个新组。
  • /lp group <group> delete: 删除一个组。

权限

  • luckperms.admin: 允许用户使用所有 LuckPerms 命令。
  • luckperms.user: 允许用户使用与自身相关的命令。
  • luckperms.group: 允许用户使用与组管理相关的命令。

示例

设置玩家权限

plaintext复制代码
/lp user Notch permission set luckperms.* true

创建新组并设置权限

plaintext复制代码
/lp group Moderator create
/lp group Moderator permission set essentials.kick true

将玩家添加到组

plaintext复制代码
/lp user Steve parent add Moderator
